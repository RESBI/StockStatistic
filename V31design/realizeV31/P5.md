# P5：Worker Agent、进程执行与 Embedded 完整链路

## 1. 阶段目标

P5 实现真实 Worker Agent 和 spawn Executor，将 P2 Kernel、P3 Storage、P4 Dispatcher 组合为 `StockStat.local()` 嵌入式单机拓扑。此阶段首次证明同步/异步都走同一 Job/Lease/Artifact 链路。

## 2. 必须范围

- Worker identity/session/register。
- 资源检测与 reservation。
- claim/backpressure。
- Lease start/renew/complete/fail/release。
- spawn Process Supervisor。
- Artifact resolver/local cache/publisher。
- Kernel capability registry/execution。
- cancellation/timeout/process crash。
- Checkpoint Artifact 基础接口。
- Embedded Control/Event/Artifact Channels。
- `StockStat.local()` 最小 SDK。
- Indicator、Backtest 同步/异步 E2E。

## 3. Worker Agent 任务

## 7. Checkpoint 基础

P5 不要求恢复完整回测，但建立真实接口：

- [ ] Executor 可发布 Checkpoint Artifact。
- [ ] Checkpoint Manifest 绑定 input digests/version/attempt。

这个测试 capability 必须位于 tests，不成为公开 `custom` 任务。

## 9. 测试计划

### 9.1 Worker unit

- identity/session。
- capability load failure 显式。
- resource reserve/release。
- claim capacity。
- heartbeat/drain。
- backoff/jitter。

### 9.2 Spawn/process

- Indicator child success。
- Backtest child success。
- child Python exception -> work.fail。
- abrupt exit/kill。
- timeout terminate。
- cancellation cooperative/forced。
- stdout/stderr capture/limit。
- temp cleanup。
- global numpy state 不泄露到下一 Attempt。

### 9.3 Artifact/cache

- 第一次下载、第二次 cache hit。
- digest mismatch 拒绝。
- cache eviction/pin。
- output upload success + complete。
- upload success/complete stale -> orphan。

### 9.4 Lease/fault

- renew 正常。
- Agent kill 后 lease expiry/retry 另一个 Agent。
- child kill 但 Agent 上报 fail。
- complete ack 丢失后重发。
- generation 1 迟到被拒绝。
- Dispatcher 暂停短于/长于 TTL。

### 9.5 Embedded E2E

- `StockStat.local()` ingest/query 或加载 fixture。
- 同步 MA 创建完整 Job/Event。
- 异步 MA + JobHandle。
- 同步 backtest 与 P2 direct Kernel parity。
- async backtest cancel。
- Local process restart 后可查询已完成 Job/result。
- 同一 DatasetSnapshot 第二个 Job 复用 Artifact/cache。

### 9.6 Windows/Linux

- spawn 是强制路径。
- 文件锁/atomic rename。
- process terminate。
- path/timezone。

## 10. 性能目标

| 场景 | 目标 |
|---|---|
| Embedded control overhead | 有测量；小任务可 batch/热进程优化，不允许旁路 |
| Worker idle | 无 busy-loop |
| 50MB input second use | Worker cache 命中，不重复 copy/download |
| CPU scaling | 2/4 process 对可并行任务有可解释加速 |
| Process leak | 测试后无残留 child |

## 12. 阶段演示

一个 Python 脚本：

```python
ss = StockStat.local("demo-v31.db")
job = ss.backtests.submit(...)
for event in job.events():
    print(event.sequence, event.event_type, event.progress)
result = job.wait().as_backtest()
print(result.metrics)
```

另一个故障演示在运行中 kill Worker Agent，启动新 Agent 后 Job 自动完成，旧 Attempt complete 被拒绝。

## 13. 风险与处理

| 风险 | 处理 |
|---|---|
| spawn 序列化上下文失败 | ExecutionContext 只用 Contracts/refs，不传 Client/DB 对象 |
| 小任务 overhead 大 | 热进程、batch、阈值由 Planner；不引入直调旁路 |
| Agent 与 child IPC 卡死 | timeout/health monitor/pipe close 测试 |
| Windows kill/cleanup 差异 | 专用 adapter + CI |
| Embedded 隐藏边界问题 | 强制 serialization contract tests |

## 14. 退出条件

- Worker 使用 spawn 进程执行 P2 能力。
- Lease/fencing/complete 重试真实接线。
- Artifact cache 与发布链路工作。
- `StockStat.local()` 同步和异步均通过完整 Job 链路。
- Worker/child crash 故障测试通过。
- Local 重启后 Job/result 可恢复。
- 无 direct Kernel bypass 和 legacy import。
