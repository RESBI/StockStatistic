# P7：复合实验、Reducer 与 N Worker

> 前置：[P6 Backtest Kernel](P6.md)
> 输出：batch/search/simulation/validation 的确定性分布式执行
> 后续：[P8 User Surface and Migration](P8.md)

## 1. 阶段目标

实现复合金融 Job 的 Planner、WorkUnit DAG、N Worker 并行和 Worker-side reducer。用户提交一个实验 Job，Dispatcher 只规划和跟踪引用，不执行 pandas 合并或算法计算。

阶段闭环：

```text
compound Job -> deterministic DAG -> N Workers -> partial manifests
-> reducer WorkUnit -> final experiment/simulation/validation manifest
```

## 2. 必需交付物

### 2.1 ExecutionPlan 与 DAG

完善：

- WorkUnit dependency edges。
- `blocked -> ready` 原子迁移。
- fan-out/fan-in。
- internal-only merge/rank operations。
- DAG cycle、node/edge/parameter expansion 上限。
- canonical unit ID/run ID/trial ID/sample ID。
- partial failure policy。
- Job progress 按 units 和 operation phase 计算。

Planner 输出必须可序列化、可 golden、与 Worker 数无关。

### 2.2 `experiment.batch@1`

输入：Snapshot、run 列表或 canonical matrix。实现：

- strategy x fee x instrument x config 展开。
- stable run ID 和去重。
- batch size 只影响 unit grouping，不影响结果顺序。
- 每 run 独立 error status。
- fail-fast 和 collect-errors policy。
- `experiment.merge@1` 输出 experiment table。
- summary、ranking 和 failed run report。

PAXG 52 x 4 使用该 operation，不由客户端循环提交 208 个顶层 Job。

### 2.3 `experiment.grid_search@1`

实现：

- parameter space schema 和 Cartesian product。
- conditional/invalid combinations 的受控过滤。
- canonical parameter order 和 trial ID。
- base BacktestParameters + overrides。
- objective metric/direction。
- single/multi objective 的首期明确范围。
- rank reducer、tie breaking 和 top-N result refs。
- max trials/quota。

### 2.4 `experiment.random_search@1`

实现固定 distribution schema 和 global seed：

- sample/trial ID 唯一。
- trial parameters 由 `H(seed, trial_id)` 产生。
- shard 数和完成顺序不影响采样集合。
- 与 grid 共用 rank reducer，不复制回测执行逻辑。

### 2.5 `experiment.bayesian_search@1`

迁移当前 Optuna 类能力，使用持久 study/trial 状态和注册 sampler，不把任意 optimizer 对象传入 Worker：

- 首批 sampler 为固定版本 TPE，参数分布使用类型化 schema。
- study seed、sampler/pruner 实现版本和 objective 定义进入 lineage。
- trial suggestion、state 和 value 持久化，Dispatcher/Worker 重启后可继续未完成 study。
- 同一 trial 只允许一个有效 attempt，重试复用其已建议参数。
- 单目标 minimize/maximize 为必需；多目标只有在 Pareto schema 和测试完整后增加。
- 支持 max trials、timeout 和 failed/pruned trial policy。
- pruner 首期只使用已注册且可重放的规则；不提供任意 callback。
- 输出完整 trial table、best trial、study summary 和每个 backtest result ref。

Bayesian search 的采样轨迹对并发完成顺序可能敏感，因此 descriptor 必须明确其确定性等级。固定 scheduler policy、sampler version 和 seed 的 reference mode 用于 parity；高并发 opportunistic mode 必须在结果 lineage 中明确标记，不伪称与串行轨迹完全相同。

### 2.6 模拟 operation

实现：

- `simulation.bootstrap_paths@1`。
- `simulation.order_shuffle@1`。
- `simulation.monte_carlo@1` 的首批经验分布方法。

确定性规则：

```text
sample_id = 0..N-1
sample_seed = H(job_seed, sample_id)
shard = a range/set of sample_id
```

输出：sample table、return/drawdown distribution、quantiles、failure count 和 source result ref。

### 2.7 Validation operation

实现：

- `validation.walk_forward@1`。
- `validation.subperiod@1`。
- `validation.regime@1`。
- `validation.sensitivity@1`。
- `validation.negative_control@1`。

要求：

- chronological window validation。
- train/test/embargo boundary schema。
- ordered merge。
- 每窗口独立 result ref。
- 防止 optimization data 泄漏到 validation。
- PAXG BTC negative control 和费率敏感性可表达。

`validation.purged_cv@1` 若实现，必须有 purge/embargo 数学和泄漏测试；不以普通 K-fold 冒充。

### 2.8 Reducer capabilities

实现 Worker-side：

- `experiment.merge@1`。
- `experiment.rank@1`。
- `simulation.quantiles@1`。
- `validation.merge@1`。
- 必要的 hierarchical merge，避免单 reducer 内存过大。

Reducer 输入只含 committed ArtifactRefs。输出仍先 commit Storage，再 complete。Dispatcher 只在依赖完成后把 reducer unit 转 ready。

### 2.9 Partial result

长实验支持：

- immutable partial manifest sequence。
- completed run/trial/sample ranges。
- partial experiment table。
- Job event `partial.available`。
- final manifest 引用或覆盖完整 logical result。
- Client 按 sequence 读取，不把 partial 当 final result。

### 2.10 N Worker 调度增强

实现：

- 同一 Job 多 unit 并行。
- per-tenant active unit quota。
- large/small Job fairness。
- reducer resource class。
- cache affinity 和 Snapshot 热点提示。
- retry 后 unit 可换 Worker/resource class。
- worker drain 下已运行任务完成、新任务停止。

## 3. PAXG 分布式验收

### 3.1 Batch

- 一个 `experiment.batch@1` Job 产生 52 x 4 = 208 runs。
- B1 买入持有、B2 周一定投、B3 价格曲线收益进入同一 comparison/report lineage。
- 1、2、4、8 Worker/slot 配置均完成。
- run table canonical 排序一致。
- 每个 run 的 backtest result ref 可单独追溯。

### 3.2 Search

- S21 threshold grid 至少复现当前 5 个 trial，用更完整空间做扩展测试。
- trial order 和 best result 不随 batch size/Worker 数改变。
- invalid combination 有明确状态。
- 增加一个固定 seed/version/reference scheduler 的 Bayesian study golden，验证重启续跑和 best trial。

### 3.3 Monte Carlo

- 当前 200 sample/4 shard 作为 smoke。
- 正式确定性测试至少 1,000 sample，运行 1/2/4/8 shards。
- sample rows、quantiles 和 drawdown summary 一致。

### 3.4 Walk-forward

- 当前 3 个 chronological windows 作为 smoke。
- 增加 train/validation split、boundary 和 ordered merge 测试。
- 1/2/4 Worker 输出顺序和结果一致。

## 4. 实现步骤

1. 完善 DAG persistence、blocked/ready 和 dependency transaction。
2. 实现 batch planner + merge reducer，先通过 208 runs。
3. 实现 grid/random planner + rank reducer。
4. 实现持久 Bayesian study/trial、注册 sampler 和 reference scheduler。
5. 实现 simulation sample-ID sharding + quantile reducer。
6. 实现 walk-forward/subperiod/regime/sensitivity/negative control。
7. 实现 partial manifest 和 SDK partial view。
8. 增强 fairness/quota/reducer resource scheduling。
9. 加入 N Worker、retry、cancel 和 reducer failure tests。
10. 完成 PAXG 分片不变性和吞吐报告。

## 5. 测试计划

### 5.1 Planner golden

- batch run IDs/order/grouping。
- grid combinations/order/trial IDs。
- random trial parameters for fixed seed。
- Bayesian reference-mode suggestions/states/best trial 和 restart continuation。
- Monte Carlo sample IDs/shards。
- walk-forward chronological windows。
- merge dependencies。
- empty space、too-large DAG、cycle、invalid override。

### 5.2 DAG 和状态

- blocked 只在全部成功依赖后 ready。
- upstream failure 的 dependent cancel/fail policy。
- retry 不创建重复 logical run。
- reducer 只执行一次有效 attempt。
- Job cancel 停止新 leases 并终止 running units。
- partial success 与 final state 一致。
- Dispatcher restart 恢复 DAG。

### 5.3 分片确定性

- 1/2/4/8 shards 的 sample/trial/run 集合相同。
- completion order shuffle 不影响 final table/digest。
- Worker loss/retry 不重复 sample ID。
- hierarchical merge 与 single merge 一致。
- seed 和 environment 进入 lineage。

### 5.4 Reducer

- 输入 manifest 缺失/未 committed 被拒绝。
- schema mismatch。
- duplicate refs 去重或明确冲突。
- reducer OOM 可换 large pool 重试。
- reducer failure 不让 Dispatcher 进程加载结果。
- result table canonical sort 和 tie breaking。

### 5.5 调度和性能

- 1 个 large Job 不饿死 small Job。
- tenant quota。
- priority aging。
- 2/4/8 Worker 有合理吞吐提升报告，不设不现实线性硬门槛。
- Storage/cache 成为瓶颈时指标可见。
- Dispatcher RSS 与实验结果大小近似无关。

### 5.6 PAXG

- 208 run 完整矩阵和 3 benchmarks。
- 代表策略结果与 P6 单 run 一致。
- threshold search、Monte Carlo、walk-forward 完整。
- 1/2/4/8 分片最终 result schema、row set 和数值一致。
- lineage 从 final report 到每个 backtest 和输入 Snapshot。

## 6. 验收门禁

1. batch/grid/random/Bayesian/Monte Carlo/walk-forward 有正式 planner 和 reducer。
2. Dispatcher 不执行结果 merge/rank/quantile 计算。
3. PAXG 208 runs 由单个 batch Job 完成。
4. PAXG search/Monte Carlo/walk-forward 在不同 Worker/分片配置结果一致。
5. partial、cancel、retry、Worker loss、reducer failure 测试通过。
6. DAG/reducer 在 Dispatcher 重启后可恢复。
7. tenant quota/fairness 不发生明显饥饿。
8. final experiment/simulation/validation manifest 有完整 lineage。

## 7. 非目标

- 不做任意用户 DAG 编辑器。
- 不做 Airflow 类通用 orchestration。
- 不实现跨 Dispatcher 联邦调度。
- 不实现可恢复运行中 checkpoint。
- 不把所有统计研究自动转换为大 DAG；轻计算可保持单 unit。

## 8. 风险与处理

| 风险 | 处理 |
|---|---|
| unit 数爆炸 | planner quota、grouping、hierarchical reducer |
| Worker 数改变随机结果 | global sample/trial ID 决定 seed，shard 只分配 ID |
| reducer 成为新单点内存瓶颈 | Worker large pool + hierarchical merge + columnar reads |
| partial 被误当 final | distinct schema/state，Job result 只指向 final committed manifest |
| 复合 operation 复制回测逻辑 | planner 始终展开 `backtest.run@1`，不内嵌第二套引擎 |
