# P7：金融复合任务与现有功能完整迁移

## 1. 阶段目标

P7 在已稳定的单次原子任务链路上迁移 StockStat 当前全部重型金融功能：参数搜索、批量回测、费率扫描、Monte Carlo 和 Walk-forward，并补齐单次回测的完整现有能力矩阵。

本阶段的关键是“Planner fan-out/fan-in + 原子 Executor 复用”，不得重新为每个复合功能复制回测引擎。

## 3. Planner 设计任务

### 3.1 Fan-out 控制

- [ ] 组合数预计算。
- [ ] max fan-out/quota。
- [ ] batch sizing heuristic。
- [ ] 目标 WorkUnit duration 配置。
- [ ] 资源估算。
- [ ] plan digest 包含候选和 seed 分配。

### 3.2 Fan-in/Reducer

- [ ] Reducer 是所属 capability 的 `executor_role=reduce` Worker 入口，不公开通用 `*.reduce` capability。
- [ ] 分层归并支持大量 shard。
- [ ] input Artifact refs 排序稳定。
- [ ] 缺失/重复 shard 检测。
- [ ] Reducer retry 幂等。
- [ ] Reducer 高内存资源 profile。

### 3.3 Failure policy

每个复合能力明确：

| 情况 | Search | Batch | Monte Carlo | Walk-forward |
|---|---|---|---|---|
| 单 trial 业务失败 | 记录 failed trial，可按阈值继续 | 默认记录并继续/可 strict | shard infrastructure retry | window strict/可配置 |
| Worker lost | retry | retry | retry | retry |
| 全部无有效结果 | Job failed | Job failed | Job failed | Job failed |
| 用户取消 | 停止新 work，已有结果按 policy 保留 partial | 同左 | 同左 | 同左 |

## 4. Determinism

### 4.1 Seed 分配

随机流不依赖 Worker ID、领取顺序或重试次数。建议：

```text
child_seed = SeedSequence(base_seed, spawn_key=(stage_index, partition_index))
```

Attempt 重试使用相同 partition seed。

### 4.2 排序

- Candidate ID 来自 canonical parameter digest。
- Trial Summary 按 candidate ID 去重。
- 相同 objective 值按 candidate ID 稳定 tie-break。
- Batch result key 不依赖完成顺序。
- Monte Carlo path columns 按 simulation index。

## 5. Checkpoint

至少真实实现：

- Grid/random candidate batch 的 completed index/checkpoint。
- Monte Carlo simulation range 内的 completed subrange。
- Reducer 可从已提交 partial Artifact 恢复。

完整单次 Backtest checkpoint 仍可不实现，但取消必须在 bar 边界响应。

## 7. 测试计划

### 7.1 Search parity

- 当前 `grid_search` 小/中网格。
- 单 Worker vs 4 Worker ranking exact。
- 不同 batch size exact。
- Worker crash/retry exact。
- tie/NaN/failing candidate。
- Optuna 固定 sampler seed 的可复现范围和 API 迁移。

### 7.2 Batch/fee parity

- StrategyBatchRunner 当前案例。
- fee_sweep/maker_taker_sweep。
- strategies x fee models 完整 key/metrics。
- 完成顺序随机不改变 summary。
- detailed Artifact lazy fetch。

### 7.3 Monte Carlo parity

- 固定 seed 单 Worker vs 多 Worker exact path/index。
- n_samples 非整除 partition。
- retry 不重复/漏 simulation index。
- summary-only 与 full paths 统计一致。
- 旧 Monte Carlo golden。

### 7.4 Walk-forward parity

- window 边界与旧实现一致。
- `[start,end)` 无重叠错误。
- step/train/test 组合。
- per-window result。
- cancel/retry。

### 7.5 Backtest full matrix

迁移当前 `test_backtest_*.py` 中的业务意图，重写到新 API/Kernel，不机械复制只验证旧类结构的测试。

### 7.6 Ingestion Job

- synthetic/yfinance/ccxt adapter contract。
- 中断后 checkpoint/resume。
- Worker/Storage 重启后幂等继续。
- progress 和 data quality event。
- 新数据写入后后续 Snapshot 使用新 watermark。
- 当前在线/离线 `ingest()` 功能迁移对比。
- on-demand/cron trigger 与 incremental ingestion 迁移、停机 catch-up 和重复触发测试。

### 7.7 PAXG

- 代表 5 策略 x 4 费率逐字段 parity。
- 完整 33 x 4 summary parity。
- 单机与多 Worker 一致。
- 记录性能和 Artifact 体积。

### 7.8 Fault/performance

- 1000 组合参数搜索 4/8 slots 加速。
- 中途 kill 25% Worker。
- Reducer crash/retry。
- partial event 和 resume。
- Dispatcher restart 在 fan-out 中途。
- 大量小 WorkUnit batch heuristic 比较。

## 8. 性能门禁

性能目标以串行 Kernel 基线动态计算，避免写死不可信绝对值：

| 场景 | 目标 |
|---|---|
| 132 回测，8 有效 CPU slots | 加速比 >= 5，且结果一致 |
| 1000 重型 grid trials，32 slots | 加速比 >= 16，调度/数据开销可解释 |
| Monte Carlo 10000 | 随 slots 有显著加速，seed exact |
| Storage 主库 | 每个共享 Snapshot 1 次查询 |
| Reducer | 不在 Dispatcher 进程显著增加 RSS |

若任务本身过轻，benchmark 必须增加数据/工作量，不能用调度 overhead 主导的 toy case 宣称失败或成功。

2. 展示 Planner Stage/WorkUnit 数和 batch size。

## 11. 风险与处理

| 风险 | 处理 |
|---|---|
| 每 trial 一个 WorkUnit overhead | candidate batching + benchmark |
| Strategy mutable state 跨 trial | 每 trial/batch 明确新 factory instance |
| Monte Carlo 并行 seed 漂移 | simulation index + SeedSequence |
| Reducer 过大 | 分层 reducer + Arrow scan |
| Partial success 语义模糊 | 每项能力 schema 固定 failure policy |

## 12. 退出条件

- 当前重型金融功能全部映射到新 capability。
- 当前数据采集功能已接入持久 Job，并通过幂等/恢复测试。
- 复合任务只复用原子 Executor，不复制回测内核。
- Search/Batch/Monte Carlo/Walk-forward 分布式 parity 通过。
- PAXG 代表和完整摘要 parity 通过。
- Worker crash/Reducer retry/Dispatcher restart 测试通过。
- 性能和带宽达到门禁或有基于数据的批准例外。
