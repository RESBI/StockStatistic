# P6：全新回测内核与 52 策略完整迁移

> 前置：[P3 Finance Basic](P3.md)、[P5 Remote Worker](P5.md)
> 输出：确定性 `backtest.run@1`、StrategyBundle 和完整 PAXG 52 x 4 parity
> 后续：[P7 Compound Experiments](P7.md)

## 1. 阶段目标

完全重写回测内核，统一时间、数据可见性、订单、执行、成本、持仓和结果语义。P6 必须消除现有 PAXG V3.1 样例的 7 个 `analysis_only` 缺口，使 52 个策略均通过正式 V3.1 Strategy API 和 `backtest.run@1` 执行。

阶段闭环：

```text
Snapshot + StrategyBundle + BacktestParameters
-> SimulationClock -> orders/fills/portfolio
-> committed BacktestResult manifest
```

## 2. 必需交付物

### 2.1 回测领域模型

实现不可变或受控模型：

- `SimulationClock`、`MarketEvent`、`SessionEvent`、`TimerEvent`。
- `DataCursor` 和 point-in-time data view。
- Strategy lifecycle/context。
- Order、OrderGroup、OCO、parent-child。
- Fill、Trade、Position、CashLedger、PortfolioLedger。
- ExecutionPolicy、CostModel、Risk/ExitRule。
- BacktestParameters 和 BacktestResult schema。

所有 enum、单位、时区和价格/数量精度明确进入契约。

### 2.2 SimulationClock

支持：

- 多标的稳定事件排序。
- parent timeframe 与 intrabar timeframe。
- 7x24 和交易日历 session。
- session open/close。
- 跨 session 持仓。
- 绝对时间定时器。
- fill 后相对持有时长定时器。
- 同时间戳稳定优先级。
- 第一根 sub-bar 关闭后策略事件。
- Job 取消检查和结构化 progress。

### 2.3 Point-in-time 防护

策略在任意事件只能看到当时已解锁数据：

- parent bar 开始时不可见该 bar high/low/close。
- intrabar 按 sub-bar 逐步解锁。
- rolling/history 查询上界为 `now`。
- 多 timeframe 数据按 release time 对齐。
- DatasetSnapshot 越界访问被拒绝。
- 指标 warm-up 不可读取未来值。

测试必须包含诱饵未来值和恶意策略访问尝试。

### 2.4 Order 生命周期

支持：

- market、limit、stop、stop-limit。
- created/active/partially filled/filled/cancelled/expired/rejected。
- GTC、DAY、IOC、FOK。
- absolute expiration。
- duration/post-fill expiration。
- OCO first-fill-cancels-other。
- mutual OCO 双向 entry。
- entry-exit group。
- trailing stop 的 peak/trough 更新、触发和 gap rule。
- partial fills 的首期明确策略。
- order priority 和同 bar 冲突。

### 2.5 ExecutionPolicy

使用一个统一抽象替代重叠的 FillModel/ExecutionModel：

- `next_bar@1`。
- `same_bar@1`，仅在可证明无 future access 时启用。
- `intrabar@1`。
- next open/close、limit/stop 触发。
- TP/SL 同 bar ambiguity：conservative、optimistic、path-defined。
- maker/taker classification。
- zero/default latency；复杂微观结构后置。
- close/session order policy。
- slippage hooks。

### 2.6 成本与组合

实现并测试：

- proportional、fixed、tiered cost。
- `cost.binance@1` spot/futures、BNB discount、maker/taker。
- configurable slippage。
- long/short。
- cash/margin 可用性和拒单。
- 多标的持仓和 mark-to-market。
- realized/unrealized PnL。
- position sizing/risk hooks。
- benchmark 和 periods-per-year 明确定义。

### 2.7 Strategy API 和 StrategyBundle

正式 API：

```python
on_start(ctx)
on_event(ctx, event)
on_fill(ctx, fill)
on_end(ctx)
```

Context 至少提供：

- point-in-time market/history access。
- submit/cancel/replace order。
- OCO/group helper。
- schedule at absolute time / after duration / session event。
- portfolio/position read view。
- deterministic strategy config。
- progress/cancellation。

StrategyBundle：

- `manifest.json`、source、lock、signature metadata。
- deterministic zip build 和 digest。
- entrypoint/API version validation。
- safe extraction。
- trusted local 自动打包。
- 远程不接受 cloudpickle closure。

### 2.8 `backtest.run@1`

实现完整 schema、descriptor 和 executor：

- 输入 SnapshotRef 和 Strategy CodeRef。
- BacktestParameters。
- capability `cpu-backtest` 和 `builtin/signed_strategy` security profile。
- deterministic seed/environment。
- progress 按时间范围或 event count。
- 结果先写 Storage。
- final summary 只含小型 metrics 索引。

### 2.9 BacktestResult 资产

结果 manifest 成员：

```text
summary.json
equity.parquet
returns.parquet
orders.parquet
fills.parquet
trades.parquet
positions.parquet
realized.parquet
benchmark.parquet
metrics.json
diagnostics.json
```

实现 `backtest.analyze@1` 的最小版本，使 metrics 可从已有结果重新计算，而不重跑策略。

### 2.10 PAXG 52 策略迁移

将策略目录的 52 个策略全部改写为正式 StrategyBundle/native strategies。特别补齐：

| Strategy | 当前缺口 | V3.1 能力 |
|---|---|---|
| S7 | Friday close entry 跨 session | session event + 跨 session position |
| S8 | Friday 到 Monday 持仓 | session event + scheduled Monday exit |
| S13 | Monday 到 Wednesday | absolute/session timer |
| S17 | first Monday sub-bar 后决策 | intrabar close event |
| S18 | exact six-hour exit | absolute/duration timer |
| S25 | conditional time stop | duration timer + open position condition |
| S46 | six-hour post-fill exit | post-fill timer |

禁止用近似 close 或跳过运行代替。

## 3. Parity 标准

### 3.1 52 x 4 矩阵

- 52 策略 x 4 Binance fee models = 208 successful runs。
- B1 买入持有、B2 周一定投、B3 价格曲线收益成功，并可与策略结果统一比较。
- 不存在 `analysis_only`、silent skip 或 substitute strategy。
- 每个 run 有 run ID、strategy digest、Snapshot digest、cost/execution version 和 result manifest。

### 3.2 代表策略

至少比较：

- S9 Range Breakout。
- S21 Extreme Reversal。
- S45 Core B Close。
- S48 Core B Profit。
- S51 Core B Aggress。

比较维度：

- total return。
- fills 和 trade count。
- entry/exit timestamp、side、price、quantity。
- fees。
- max drawdown。
- representative equity points。

Sharpe 若旧实现使用不同采样频率，V3.1 采用唯一公式，并输出 migration difference，不人为匹配旧错误定义。

### 3.3 新补齐策略

7 个原缺口策略必须同时与 v5-redo 行为基线和手工构造的小 fixture 对比。优先验证时间和 session 语义，不只比较总收益。

1. 定义时钟、事件、point-in-time view 和基础订单模型。

10. 增加 lookahead、property、platform 和 failure tests。

## 5. 测试计划

### 5.1 时钟与可见性

- single/multi instrument deterministic order。
- parent/intrabar release sequence。
- Friday/Monday/Wednesday session event。
- exact six-hour timer，跨日/UTC。
- post-fill timer 从真实 fill time 计算。
- DST calendar fixture。
- future high/low/close 诱饵不可见。

### 5.2 订单与执行

- 每种 order type 的触发/拒绝/过期。
- gap through limit/stop price rule。
- partial fill rule。
- OCO 和 mutual OCO。
- trailing stop 的更新、触发、gap 和 short-side 对称性。
- TP/SL 同 bar conservative/optimistic。
- order priority 和 same timestamp。
- cancel/replace。
- maker/taker 分类。

### 5.3 组合与成本

- long/short PnL。
- insufficient cash/rejected order。
- fixed/proportional/tiered/Binance fees。
- BNB discount。
- slippage 对 entry/exit/fee 的影响。
- multi-asset equity。
- realized/unrealized reconciliation。

### 5.4 属性和确定性

- 相同输入/seed/environment 结果 digest 相同。
- input order 不影响稳定事件结果。
- Worker 数不影响单 backtest。
- equity end = initial + realized/unrealized - costs 的会计恒等式。
- no fills 时 metrics 稳定。
- cancel 中断不发布成功 manifest。

### 5.5 Bundle 与安全

- deterministic bundle digest。
- invalid entrypoint/API/digest/signature。
- zip slip、symlink、zip bomb 限制。
- remote closure/cloudpickle 被拒绝。
- strategy workspace 隔离和无特权 hooks。

### 5.6 PAXG parity

- 52 strategy migration status 全部 native。
- 208 runs 和 3 benchmarks 全部成功。
- 代表策略逐 fill parity。
- 7 个补齐策略时间行为 parity。
- LocalWorker/RemoteWorker 结果一致。
- Windows/Linux 数值容差和 environment digest。

## 6. 验收门禁

1. `backtest.run@1` 在本地和远程正式执行链可用。
2. point-in-time/lookahead 测试全部通过。
3. market/limit/stop/stop-limit、OCO、intrabar、session 和 timer 语义完整。
4. StrategyBundle 可确定构建、验证和执行，远程无 cloudpickle。
5. 52 个 PAXG 策略全部 native，7 个旧缺口关闭。
6. 52 x 4 = 208 runs 和 3 benchmarks 全部成功并有 lineage。
7. S9/S21/S45/S48/S51 在定义容差内 parity。
8. Local/remote 和 Windows/Linux 结果符合确定性约束。
9. BacktestResult 各成员可独立读取和重新分析。

## 7. 非目标

- 不实现 L2/L3 order book 和真实排队模型。
- 不实现任意经纪商实时交易。
- 不实现可恢复 checkpoint/preemption。
- 不支持运行时公网 `pip install`。
- 不实现完整保证金、清算和杠杆交易所模拟，除非 PAXG parity 明确需要。
- 不在内核启动分布式并行。

## 8. 风险与处理

| 风险 | 处理 |
|---|---|
| 回测语义一次性过大 | 按 clock -> order -> intrabar -> timer 垂直递增，每步 golden |
| 为 parity 复制旧 bug | 先定义 V3.1 唯一语义，差异单列；交易时序必须可解释 |
| 7 个策略只做特判 | 用通用 session/timer API 实现，不按策略 ID 写内核分支 |
| bundle 安全被低估 | P6 完成格式和隔离 hook，P9 做生产 sandbox 门禁 |
| 结果只比总收益掩盖错误 | 代表策略逐 fill/order/timestamp 比较 |
