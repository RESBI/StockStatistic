# P3：Storage、Dataset Snapshot 与 Artifact 数据面

## 1. 阶段目标

P3 实现 V3.1 独立 Storage 角色，打通“可变市场数据库 -> 不可变 DatasetSnapshot -> ArtifactRef”链路，并提供本地与生产可替换的 Artifact Store 合同。

P3 结束时尚无真实 Dispatcher/Worker，但可由测试代码创建 Snapshot、上传 Kernel 结果、下载并校验 Artifact。

## 2. 必须范围

- Instrument/Data catalog。
- OHLCV SQLite/PostgreSQL repository 合同。
- 现有 SQLite 数据迁移。
- yfinance/binance/coinbase/synthetic adapter 的新边界。
- Ingest batch 与数据质量记录。
- DatasetSnapshot service。
- Artifact metadata。
- Local filesystem Artifact store。
- S3-compatible adapter 合同和容器集成测试。
- create/upload/commit/download/GC。
- Storage control/data API。

## 8. 测试计划

### 8.1 Repository contract

同一测试参数化 SQLite/PostgreSQL：

- upsert 幂等。
- range/filter/cursor。
- 事务 rollback。
- concurrent read/write。
- timezone。
- uniqueness。

### 8.2 Legacy migration

- 复制旧 fixture DB 后 dry-run。
- 迁移后行数/范围/digest。
- 重复迁移幂等。
- 无效 symbol/source 报告。
- 源 DB hash 不变。

### 8.3 Snapshot

- 1KB/50MB/500MB synthetic dataset。
- 主 repository query 调用次数断言为一次。
- Arrow row batch 输出。
- digest 稳定。
- watermarks 变化生成新 snapshot。
- 并发相同 snapshot 请求只发布一个有效 Artifact。
- 构建中崩溃不暴露半成品。

### 8.4 Artifact adapters

- LocalFS 与 MinIO contract suite。
- multipart/Range。
- commit 重试。
- digest mismatch。
- metadata 存在/blob 缺失。
- blob 存在/metadata 缺失。
- GC 和 active pin。

### 8.5 API

- content type、大小限制和错误码。
- 大 Artifact response streaming，内存不随文件等量增长。
- 未授权路径预留测试 hook，完整 auth 在 P9。
- path traversal 和任意 local path 拒绝。

### 8.6 Kernel interoperability

- P2 MarketDataset 可直接读取 Snapshot Arrow。
- P2 Backtest output 可作为 Artifact commit 并重新下载解析。
- Artifact schema_ref 与 Kernel Result Manifest 匹配。

## 9. 性能与带宽门禁

| 场景 | 目标 |
|---|---|
| 50MB Snapshot | 单次主库 query，分块写，峰值内存有界 |
| 32 consumer | 不触发 32 次主库 query |
| cache hit | 不重建 Snapshot |
| LocalFS read | 支持 mmap/Range |
| MinIO | 可并发下载、摘要一致 |

P3 不要求 Worker cache 命中；该项在 P5/P6 验证。

## 11. 阶段演示

1. 将旧 SQLite fixture 迁移到新 Storage。
2. 请求 BTC 1h 一年 Snapshot。
3. 显示 Snapshot Manifest、row count 和 digest。
4. 用 P2 Kernel 运行回测并上传结果 Artifact。
5. 下载 equity Artifact 并校验 digest/Arrow schema。

## 12. 风险与处理

| 风险 | 处理 |
|---|---|
| Snapshot 查询锁住写入 | 使用数据库一致性策略和批次边界；实测 SQLite/PG |
| 对象与 metadata 双写不一致 | 临时 upload + 原子 publish + reconciliation |
| 50MB+ 内存复制 | Arrow RecordBatch、streaming hash |
| source adapter 行为漂移 | adapter contract + frozen synthetic fixture |
| 旧 symbol 歧义 | dry-run 报告并要求明确 venue/source |

## 13. 退出条件

- SQLite/PostgreSQL repository 合同通过。
- 旧数据库迁移器可生成完整报告并通过 fixture 验证。
- DatasetSnapshot 可生成、复用、追溯且不可变。
- LocalFS 和 MinIO Artifact adapter 合同通过。
- 大数据不进入控制 JSON。
- 主数据库一次快照查询目标有自动测试。
- P2 Kernel 与 Snapshot/Artifact 完整互操作。
