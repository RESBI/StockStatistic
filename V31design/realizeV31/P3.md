# P3：Finance 基础计算与研究能力

> 前置：[P1 Contracts](P1.md)
> 可并行依赖：[P2 Storage](P2.md) 的 Arrow/Parquet schema
> 输出：独立 `stockstat-kernel` 基础能力和首批 capability executors
> 后续：[P4 Durable Local Jobs](P4.md)、[P6 Backtest Kernel](P6.md)

## 1. 阶段目标

建立网络和调度无关的确定性 Finance Kernel，完成市场表校验、数据准备、当前指标、统计推断、路径/频域/非线性研究计算和结构化结果。P3 不实现回测执行循环和复合实验调度。

阶段闭环：

```text
MarketTable/FeatureTable -> typed operation -> deterministic result table/manifest payload
```

## 2. 包与能力边界

创建：

```text
packages/kernel/
packages/capabilities/basic/
packages/capabilities/statistics/
packages/capabilities/signal/
tests/kernel/
tests/parity/research/
```

Kernel 提供纯计算函数和金融数据模型。Capability 包把 Kernel 函数适配为 operation executor/descriptor，不访问 Dispatcher。Storage I/O 由 Worker runtime 在 P4/P5 注入。

## 3. 必需交付物

### 3.1 市场和特征数据模型

实现：

- `InstrumentId`、timeframe 和 UTC 规则。
- `MarketTable`、`FeatureTable`、`EventTable` 的 Arrow schema validation。
- 长表分组和稳定排序。
- 多标的、多 timeframe 选择。
- 交易日历和 session 的最小接口，为 P6 扩展预留。
- 显式 `.to_pandas()` 适配，不让 pandas 成为跨模块契约。
- 缺失值、quality flag、单位和列 metadata 传播。

### 3.2 数据准备 operation

实现首批：

| operation | P3 范围 |
|---|---|
| `dataset.align@1` | 多标的时间轴和 timezone/calendar 对齐 |
| `dataset.resample@1` | OHLCV 金融聚合 |
| `dataset.join@1` | 受控 as-of/exact 时间连接 |
| `dataset.window@1` | rolling、event window、weekend-Monday 配对 |
| `dataset.quality@1` | Kernel 侧计算前质量断言 |
| `feature.transform@1` | return/log return/diff/lag/normalize/rolling |

重采样规则明确 open=first、high=max、low=min、close=last、volume=sum，并处理空桶、session 和 `[start,end)` 边界。

### 3.3 当前经典指标

先从现有公开指标清单建立 definition table，逐项迁移至少包括：

- MA、EMA、MACD。
- RSI、KDJ。
- ATR、Bollinger Bands。
- returns/log returns。
- Beta。
- Sharpe、Sortino、Calmar。
- historical VaR/CVaR。
- max drawdown。
- 当前仓库其余已公开指标，按源码和 README 功能矩阵逐项核对。

每个指标记录：

- 公式版本。
- 输入列和单位。
- warm-up/min periods。
- 缺失值规则。
- population/sample 方差约定。
- annualization 和 risk-free 约定。
- 与旧实现的有意差异。

### 3.4 PAXG 特征

实现可复用 feature operation，不在研究脚本复制公式：

- x1 weekend return。
- x2 full OLS slope。
- x3 middle-window OLS slope。
- x4 weekend range。
- x5 realized volatility。
- x6 volume ratio。
- Monday max gain/loss、gap、full-day return、intraday volatility、range。
- high/low hour、up-first、first extreme 和 timing/window 类别。
- rolling 52-week、subperiod、regime 和 quintile labels。

`dataset.window@1` 必须能从 P2 Snapshot 产生 307 行研究配对表，不依赖 pickle 字典。

### 3.5 经典统计

实现：

- `statistics.describe@1`：描述、相关矩阵、分组和 rolling summary。
- `statistics.hypothesis@1`：Pearson、Spearman、one-sample/paired/independent t、Welch、chi-square、KS、Chow。
- `statistics.regression@1`：OLS、稳健标准误、rolling/segmented regression。
- `statistics.multiple_testing@1`：Bonferroni、Holm、BH-FDR。
- `resampling.permutation@1`：固定 seed、双侧/单侧、可确定 sample ID。
- `resampling.bootstrap@1`：percentile，BCa 可在有测试支持时加入。
- `statistics.survival@1`：Kaplan-Meier、log-rank 和风险比的首批需求。

统一统计结果表包含 test ID、method、statistic、p、adjusted p、effect size、CI、n、seed 和 warnings。

### 3.6 频域和非线性

实现：

- `feature.spectral@1`：CWT、Welch PSD、频带能量、peak frequency、spectral entropy、coherence。
- `feature.nonlinear@1`：mutual information、binned transfer entropy、Hurst/DFA、sample entropy、permutation entropy、简化 RQA、grey relation、GM(1,1)。
- K-means/hierarchical clustering 只在 PAXG 研究有明确输出时作为受控 method，不扩展成通用 ML API。

依赖策略：

- scipy + scikit-learn 为基础可选组。
- PyWavelets 为 spectral capability 依赖。
- Hurst/DFA、grey、GM(1,1)、TE 和简化 RQA 使用有充分测试的内置实现。
- 不依赖当前已知兼容性不稳定的 nolds 或必须 JVM 的 PyRQA 作为首期必需依赖。
- 缺失可选依赖时 capability scanner 不注册不完整 operation。

### 3.7 受控预测验证

当前 PAXG v7 已使用随机森林和时间前向切分检验多模态特征是否具有样本外增量信息，因此 P3 必须迁移这项现有研究能力。实现有限的 `validation.predictive@1`，而不是提前建设通用 ML 平台：

- 输入为 FeatureTable、目标列、明确 feature 列和 chronological split。
- 首批内置方法仅包括 linear baseline、random forest regression 和 random forest classification。
- 参数 schema 固定 n_estimators、max_depth、min_samples_leaf、seed 等受控字段。
- 缺失值处理只在训练集拟合，并应用于测试集，禁止全样本预处理泄漏。
- 输出 train/test metrics、predictions、feature importance、split metadata 和 warnings。
- 不输出可部署推理服务，不接受任意 estimator/callable，不做超参数自动优化。

该 operation 用于研究验证，与 [泛化边界](../designV31/DESIGN_GENERALIZE.md) 中未来完整 `ml.train/ml.predict` 平台分开。

## 4. PAXG 研究 parity

从 P2 Snapshot 构造 PAXG 原生研究流水线，覆盖：

| 研究版本 | 必需复现 |
|---|---|
| v1 | selected-extreme 选择偏差相关性 |
| v2 | independent gain/loss 相关性 |
| v3 | path-order chi-square 和 first-move |
| v4 | permutation、bootstrap、Chow、rolling、regime、quintile、BTC negative control |
| v6 | timing window、生存分析、多重校正 |
| v7 | CWT、PSD、spectral entropy、grey/GM、MI/TE、Hurst、entropy、RQA |

v7 还必须复现其 F 融合的 chronological holdout：受控随机森林回归的 train/test `R^2`、分类准确率和 feature importance。它属于现有研究 parity，不代表 V3.1 首期开放通用 ML training API。

固定最低 golden：

- sample count = 307。
- x1 selected-extreme `r` 约 0.594。
- x1 independent gain `r` 约 0.123。
- x1 loss `r` 约 -0.053。
- path chi-square p 值约 0.692/0.699/1.000。
- x4 -> Monday range `r` 约 0.415。
- x5 -> Monday range `r` 约 0.395。
- spectral entropy -> range `r` 约 0.041。
- v7 random forest train `R^2` 约 0.634，test `R^2` 约 0.074。
- v7 three-class test accuracy 约 0.358。

精确容差按方法记录。随机检验还要固定 seed、次数和方法版本，不能只比较宽泛区间。

## 6. 测试计划

### 6.1 数学单元

- 教科书序列和手工计算结果。
- 常数、空、单元素、短窗口和全 NaN。
- price scale、shift 和 return invariance 属性。
- white noise DFA 约 0.5，趋势序列更高。
- identical grey sequence relation = 1。
- exponential sequence GM(1,1) 误差门限。
- independent TE 接近 0，耦合序列更大。
- sample/permutation entropy 与参考实现对比。
- CWT/PSD 能识别已知周期。

### 6.2 数据语义

- UTC、DST、session 和 timeframe 边界。
- OHLCV resample 聚合正确。
- 多标的无交叉污染。
- as-of join tolerance 和 direction。
- rolling/window 不读取未来行。
- weekend-Monday 配对处理缺 bar 和 holiday。

### 6.3 统计属性

- seed 重复得到相同 permutation/bootstrap 结果。
- sample ID 分片后合并与单次执行一致，为 P7 预留。
- p-value、effect size、CI 和 n 一致。
- multiple testing 排序不影响结果映射。
- insufficient sample 返回结构化 warning/error。

### 6.4 预测验证

- chronological split 不打乱时间顺序。
- imputer/preprocessor 只用训练区间拟合。
- 相同 seed 和 sklearn/implementation version 结果一致。
- 目标列或未来测试标签不能进入 feature matrix。
- random forest train/test metrics 与独立 reference fixture 一致。
- 不允许 arbitrary estimator、pickle model 或 callable 参数。

### 6.5 Parity

- 当前经典指标与旧实现逐项 golden。
- 有意定义变化单独测试，不要求错误公式 parity。
- PAXG v1-v7 结果表和摘要 golden。
- PAXG v7 F 融合 train/test `R^2`、分类准确率和 importance golden。
- Windows/Linux 浮点容差。
- pandas 输入和 Arrow 输入数值等价。

### 6.6 架构边界

- Kernel 不 import httpx/FastAPI/SQLAlchemy/Dispatcher/Storage。
- executor 不启动线程或进程池。
- operation 不读取全局 runtime 配置。
- 不接受任意 callable 作为生产参数。

## 7. 验收门禁

1. 经典指标功能矩阵全部有 V3.1 定义和测试。
2. 数据准备与统计 operation 的 schema、descriptor 和 executor 完整。
3. PAXG 307 行 feature table 可从 Snapshot 重建。
4. v1-v4、v6、v7 关键统计在记录容差内通过。
5. v7 chronological predictive validation 在记录容差内通过且无预处理泄漏。
6. spectral/nonlinear capability 在依赖齐全和缺失两种环境表现正确。
7. 所有随机 operation 固定 seed 可复现。
8. Kernel wheel 独立安装，不依赖服务包或旧 runtime。
9. 结果可编码为 P2 可提交的 Artifact schema。

## 8. 非目标

- 不实现 backtest event loop、orders、fills 或 portfolio。
- 不实现 Dispatcher planner 和 Worker 并行。
- 不实现通用 ML 训练/推理平台；只迁移 PAXG v7 所需的受控预测验证。
- 不追求完整 RQA/TE 学术算法库；首期明确方法版本和适用边界。
- 不将研究脚本的任意 pandas 操作开放成通用 transform language。

## 9. 风险与处理

| 风险 | 处理 |
|---|---|
| 旧指标公式不一致 | 建 definition table，区分 bug parity 与语义迁移 |
| 可选科学库平台差异 | capability 自检、版本锁和 reference fixture |
| PAXG 公式再次散落 | 所有公共公式进入 Kernel operation，研究脚本只编排 |
| 浮点 golden 过脆 | 每种 method 定义合理绝对/相对容差和环境摘要 |
| 把 transform 做成任意表达式 | 仅实现有明确金融语义的 method ID |
