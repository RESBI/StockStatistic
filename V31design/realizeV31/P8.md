# P8：完整 SDK、CLI、DSL、Admin 与迁移

> 前置：[P7 Compound Experiments](P7.md)
> 输出：全部用户功能的公开入口、迁移工具和原生 PAXG 工作流
> 后续：[P9 Hardening and Cutover](P9.md)

## 1. 阶段目标

完成用户可见产品面，使当前 Python、CLI、DSL、图表、管理和研究脚本功能全部能通过 V3.1 公共 API 使用。本阶段迁移功能，不保留旧运行时 facade。

阶段闭环：

```text
user intent -> SDK/CLI/DSL -> JobHandle -> typed result/render/report
migration scanner -> actionable report -> native V3.1 code
```

## 2. 必需交付物

### 2.1 统一 SDK facade

完成：

```python
StockStat.local(...)
StockStat.connect(...)
StockStat.from_profile(...)
```

领域 API：

- `session.market`。
- `session.features`。
- `session.statistics`。
- `session.strategies`。
- `session.backtests`。
- `session.experiments`。
- `session.simulations`。
- `session.validation`。
- `session.render`。
- `session.reports`。
- `session.cluster`。
- `session.query`，仅为 DSL/typed query 前端。

Local/Remote 实现相同 public protocol，不通过 `isinstance` 分叉金融行为。

### 2.2 Table、Snapshot 和 Result views

实现用户类型：

- `MarketTable`、`FeatureTable`、`ExperimentTable`。
- `DatasetSnapshotView`。
- `ArtifactView`。
- `JobHandle`、`JobResult`。
- `BacktestResultView`、`SimulationResultView`、`ValidationResultView`、`ReportView`。

提供：

- selected member lazy download。
- `.to_pandas()`、`.to_arrow()`、`.download()`。
- `.lineage()`。
- kind-safe `.as_*()`。
- 本地缓存和显式关闭资源。

不返回任意 pickle Python object。

### 2.3 市场和研究 API

公开覆盖：

- 四数据源 ingest。
- instrument/coverage/query/snapshot。
- from pandas/Arrow/Parquet。
- align/resample/window/join/quality。
- 全部经典 indicators。
- PAXG path/spectral/nonlinear features。
- describe/hypothesis/regression/resampling/multiple testing/survival。
- table export。

每个 API 只负责编译已注册 operation JobSpec，不复制算法。

### 2.4 Backtest 和实验 API

公开覆盖：

- StrategyBundle build/ref/inspect。
- single backtest/analyze/compare。
- batch/fee sweep。
- grid/random/Bayesian search。
- bootstrap/order shuffle/Monte Carlo。
- walk-forward/subperiod/regime/sensitivity/negative control。
- 受控 chronological predictive validation，用于 PAXG v7 样本外检验。
- Job partial、cancel 和 result member access。

### 2.5 Render 和 report

实现 operation：

- `render.chart@1`。
- `render.dashboard@1` 的有限 profile。
- `export.table@1`。
- `report.assemble@1`。

首批 chart profiles：

- OHLCV + indicator。
- correlation/heatmap。
- rolling statistic。
- backtest equity/drawdown/trades。
- experiment ranking/fee matrix。
- Monte Carlo distribution。
- walk-forward windows。

ChartSpec 与 PNG/SVG/HTML 输出作为 Artifact。现有 `test_dashboard.png`、`test_equity.png` 只作为历史视觉参考，不作为唯一 golden。

### 2.6 CLI

实现：

```text
stockstat profile ...
stockstat market ingest/list/coverage/snapshot
stockstat operation list/show
stockstat job submit/status/list/watch/cancel
stockstat result inspect/download/lineage
stockstat backtest run
stockstat experiment batch/grid
stockstat cluster workers/capabilities
stockstat migrate scan
stockstat local start/stop/status
```

CLI 使用 SDK；支持 JSON 输出、非交互退出码和 human-readable 表格。不得另写 HTTP 协议实现。

### 2.7 DSL

保留有限金融 DSL：

```text
DSL text -> AST -> typed dataset/feature plan -> JobSpec
```

首期：

- market source、字段选择、时间过滤。
- indicator 调用和别名。
- resample/window 的有限语法。
- explain/compile 到 JobSpec。
- 明确禁止任意 SQL table、DDL/DML、Python、shell、网络函数。

DSL parser/compiler 与旧 DSL 语法是否逐句兼容不作为目标；提供迁移示例和 scanner 提示。

### 2.8 Admin UI

实现小而明确的只读管理界面：

- Job 列表、详情、事件和错误。
- Worker、pool、capability、slot 和 last seen。
- Snapshot/Artifact、lineage、retention 和 storage usage。
- ingest coverage 和 data quality。
- operation discovery。

可允许具有 scope 的 cancel/drain 管理动作，但 UI 只调用公开 API，不访问服务数据库。保持当前项目视觉语言时优先复用；若完全新建，设计为信息密集的运维研究台，不做通用卡片堆叠。

### 2.9 配置和认证体验

实现统一 TOML profile：

- local runtime dir。
- dispatcher/storage URL。
- token env ref。
- TLS verify/CA path。
- request timeout、download cache。
- no secret written to report/log。

支持环境变量 override 和配置诊断命令。

### 2.10 迁移 scanner

扫描 Python/配置/命令：

- `StockStatClient`、`V2Client`。
- `ComputeBackend`、`client.compute.remote`。
- `TaskSpec`、`ComputeSpec`、`Envelope`。
- `BacktestEngine`、strategy decorator/closure。
- `StrategyBatchRunner`、`grid_search`、`monte_carlo_equity`、`walk_forward`。
- cloudpickle strategy refs。
- 旧 CLI/config。

输出：

- file/line/symbol。
- severity 和是否自动可迁移。
- V3.1 target API。
- StrategyBundle/time/session manual work 提示。
- CSV、JSON 和 Markdown report。

scanner 默认只读，不自动改复杂金融逻辑。

### 2.11 文档和示例

重写：

- 根 README/README_CN 的 V3.1 候选版。
- installation/deployment。
- quickstart local/remote。
- data、features、statistics、backtest、experiments、results。
- Strategy API 和 bundle。

示例必须在 CI 执行，不能只做未验证代码块。

### 2.12 原生 PAXG 目录

创建最终研究目录，例如：

```text
working/PAXG-Weekend-Monday-Law-v31-native/
```

要求：

- 只 import `stockstat` 和公开 contracts 类型。
- 不 import Kernel internal、service package 或旧研究 Python 模块。
- 通过 Snapshot/Artifact 使用输入。
- v1-v7 research、52 x 4 batch、search、Monte Carlo、walk-forward、charts/report 全部可运行。
- 生成 manifest-based report 和完整 lineage。
- scanner 对目录返回 0 legacy findings。

## 3. 功能迁移矩阵

建立机器可检查的 `migration_matrix`，至少包含：

| 旧能力 | V3.1 入口 | 测试 |
|---|---|---|
| ingest/query/offline | market + local profile | local/remote fixture |
| indicators | features.indicators | math parity |
| DSL | query/DSL compiler | AST/JobSpec golden |
| backtest | backtests.run | P6 parity |
| batch/fee sweep/benchmarks | experiments.batch/backtests.compare | 208 PAXG runs + 3 benchmarks |
| grid search | experiments.grid_search | deterministic trials |
| Optuna/Bayesian search | experiments.bayesian_search | persistent study/trial reference mode |
| Monte Carlo | simulations.* | shard parity |
| walk-forward | validation.walk_forward | window parity |
| plots/dashboard | render/report | chart schema/golden |
| cluster info | cluster | API/CLI/Admin |
| TUI | CLI watch/Admin replacement or explicit removal decision | documented |

任何当前公开功能必须有目标入口、测试和文档，或有明确移除理由；不能因没有迁移而静默丢失。

## 5. 测试计划

### 5.1 SDK

- Local/Remote 生成相同 JobSpec digest。
- sync helper = submit + wait。
- lazy member download。
- result kind safety。
- events/partials reconnect。
- cancellation/timeouts。
- config/profile/env override。
- token 不进入 repr/log。

### 5.2 CLI

- fake Session 单元测试。
- local 和 remote smoke。
- JSON output schema 和 exit code。
- Ctrl+C 对 watch/submit 的行为。
- 下载文件 digest 校验。
- 无 TTY 环境。

### 5.3 DSL

- parser/AST/JobSpec golden。
- type/schema errors 带 source span。
- no arbitrary table/SQL/Python/shell/network access。
- query range 和 indicator semantics。
- local/remote compile 相同。

### 5.4 Render/Admin

- ChartSpec schema。
- deterministic SVG/结构 golden；PNG 只做容差或 smoke。
- selected result member loading。
- Admin tenant/role visibility。

- UI 不访问内部 DB route。

### 5.5 Migration

- 代表旧脚本手工迁移并运行。
- migration matrix 无未决公开功能。

### 5.6 文档

- Markdown link check。
- code block smoke tests。
- OpenAPI/schema links。
- local and compose quickstart from clean environment。
- 中文/英文核心命令一致。

## 6. 验收门禁

1. 当前公开功能矩阵全部有 V3.1 实现、测试和文档或明确移除决定。
2. Local/Remote 用户业务代码除连接配置外一致。
3. SDK/CLI/DSL/Admin 均只使用公开 service APIs。
4. PAXG native 全流程完成，52 x 4、search、MC、walk-forward、charts/report 齐全。
5. PAXG native 不 import 旧包、Kernel internal 或 service internal。
6. scanner 对旧 API corpus 达到约定发现率，对 native 目录为 0。
7. README/quickstart/examples 在干净环境通过。
8. 迁移指南说明所有不兼容点和定义差异。

## 7. 非目标

- 不保留长期旧客户端 shim。
- 不自动重写复杂策略。
- 不让 DSL 变成通用 SQL 或脚本语言。
- 不在 Admin 实现完整研究 notebook/IDE。
- 不要求首次切换就提供所有未来量化领域 UI。
- 不以 TUI 的形式兼容为硬目标；必须明确替代或移除。

## 8. 风险与处理

| 风险 | 处理 |
|---|---|
| SDK convenience 绕过 Job 路径 | contract test 断言 helper 生成相同 JobSpec |
| 迁移 scanner 被误认为自动迁移器 | 输出建议和报告，复杂逻辑保持人工迁移 |
| DSL 再次成为平行执行引擎 | 只编译 typed plan/JobSpec，不执行算法 |
| Admin 耦合内部数据库 | 只使用公开 API，并在测试阻断内部 route/import |
| 文档示例漂移 | executable docs 和 CI smoke |
