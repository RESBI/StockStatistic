# P8：公共 SDK、DSL、可视化与客户代码迁移

## 1. 阶段目标

P8 完成面向用户的 V3.1 产品层：一个 `StockStat` Session、金融命名空间 API、DSL 编译器、CLI、策略打包、迁移扫描、结果物化和可视化。现有 README/USAGE/PAXG 客户代码必须有完整新版本。

P8 不提供旧 `StockStatClient/V2Client/ComputeBackend` 兼容实现。

## 2. 公共 SDK

- [ ] `StockStat.connect(url, token=...)`。
- [ ] 配置文件/env/runtime 分层，但每个应用 schema 独立。
- [ ] sync/async context manager。

## 4. Indicator facade 生成

避免再次手写 23 个 facade 与 catalog 漂移：

- [ ] 从 IndicatorDescriptor 生成/绑定方法签名策略。
- [ ] 静态类型 stub/codegen 或明确通用 `compute(name, ...)` + 常用手写 facade。
- [ ] DSL function table 来自同一 descriptor。
- [ ] CLI indicators list 来自同一 descriptor。
- [ ] docs reference 自动生成。

必须防止 V2 的 direct methods、Plugin wrapper、DSL dict 三份来源重现。

## 5. DSL

### 5.1 Compiler

- [ ] 统一 parser/AST，删除旧 fallback Evaluator 概念。
- [ ] source -> DatasetSelector。
- [ ] indicator function -> capability/operation。
- [ ] type/arity/parameter validation 来自 descriptor。
- [ ] diagnostics 含行列位置。
- [ ] 编译为 Data query 或 JobSpec。
- [ ] explain 命令输出计划摘要。

### 5.2 Scope

迁移现有 SQL-like SELECT/ohlcv/WHERE/LIMIT/指标函数。复杂跨任务编排不进入 DSL 首版。

### 6.2 明确诊断

扫描并报告：

不静默退回 cloudpickle。

## 7. Migration tooling

### 7.1 Scanner

```text
stockstat migrate scan PATH
```

识别：

- `StockStatClient`。
- `V2Client`。
- `compute_backend`。
- `compute.remote`。
- 旧 `_core/_api/_domain` 内部 imports。
- `client.compute.<indicator>`。
- `client.backtest`。
- `run_dsl`。
- `MemoryStorage/SQLStorage`。
- 旧策略 decorator/closure。
- `BacktestResult.render`。
- 旧环境变量/CLI。
- 旧 on-demand/cron trigger 与 incremental ingestion Scheduler 用法。

输出机器可读 JSON + Markdown 报告：位置、替换建议、是否可自动迁移、风险。

### 7.2 Codemod

只自动修改高置信项目：

- Client 构造。
- 命名空间方法。
- TaskRef -> JobHandle 常用方法。
- offline -> local。

策略闭包和内部 imports 不自动猜测，生成手工任务。

### 7.3 Verify

```text
stockstat migrate verify OLD_SCRIPT NEW_SCRIPT
```

在固定 fixture 下运行旧/新实现，按 capability comparison policy 生成差异报告。

## 14. 阶段演示

现场迁移一个旧研究脚本：

1. `stockstat migrate scan`。
2. 自动 codemod Client/indicator/backtest 调用。
3. `stockstat strategy package` 处理策略。
4. Local 运行并 verify old/new。
5. 仅修改 Session 构造后在 remote cluster 运行。
6. CLI/Admin 查看 Job 和结果。

## 15. 风险与处理

| 风险 | 处理 |
|---|---|
| API 为迁移过度复刻旧结构 | 保持金融友好但统一 namespace/Job 语义 |
| 动态 indicator facade 类型差 | codegen stubs + descriptor source of truth |
| Notebook 策略迁移困难 | template/extraction + explicit diagnostics |
| 文档与代码漂移 | docs executable tests |
| Codemod 误改 | AST/CST 工具、高置信范围、dry-run diff |

## 16. 退出条件

- 一个 `StockStat` 公共 Session 覆盖 Local/Remote。
- 当前所有公开功能有新 API 和文档。
- DSL 无双 Evaluator/fallback。
- 策略远程执行走签名包，不走 cloudpickle。
- Migration scanner/codemod/verify 可用。
- README/USAGE/PAXG 客户代码迁移并通过 parity。
- V3.1 SDK 不导入旧实现。
