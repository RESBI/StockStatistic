# P2：Finance Kernel 与首批金融结果合同

## 1. 阶段目标

P2 建立一个纯金融、无网络和无数据库依赖的 `stockstat-kernel`，迁移现有指标和单次回测核心语义，形成后续 Worker 唯一调用的实现。

P2 的成功标准不是把旧文件搬到新目录，而是消除三套平行能力：直接函数、V2 Plugin wrapper、V3 TaskHandler。

## 2. 范围

必须完成：

- Canonical MarketDataset/Universe。
- 单一 Indicator Catalog。
- 现有 23 指标。
- BacktestEngine 单次运行核心。
- Strategy/Order/Broker/Portfolio。
- 8 成本模型、7 fill、2 execution。
- 多标的、多 timeframe、intrabar、lookahead audit。
- Backtest Result Manifest 和 Arrow schema。
- `finance.indicator.compute@1.0` Executor。
- `finance.timeseries.analyze@1.0` Executor。
- `finance.backtest.run@1.0` Executor。

暂不完成：

- Dispatcher Planner。
- Worker Agent。
- Grid/batch/Monte Carlo/Walk-forward 编排。
- HTTP、Storage snapshot。
- 公共 SDK facade。

## 4. 策略迁移原型

P2 至少支持：

| Strategy kind | P2 状态 |
|---|---|
| Builtin test strategy | 完整支持 |
| 本地 Python module factory | Kernel 测试支持 |
| 签名 wheel/source bundle | 只定义 Manifest 与 loader interface，真正沙箱在 P5 |
| Declarative strategy | 仅预留 schema，不实现语言 |

旧 `@strategy` fixture 转为模块级 factory，禁止使用 cloudpickle 作为 parity 测试捷径。

## 5. 数值兼容策略

### 5.1 指标

| 类型 | 比较 |
|---|---|
| MA/EMA/returns 等 | index/dtype 规范后 exact 或 `1e-12` |
| scipy/熵/小波 | 按现有算法设置 `rtol/atol` |
| tuple/matrix 输出 | shape、labels、每元素 tolerance |
| NaN warm-up | 位置必须一致 |

### 5.2 回测

比较字段：

```text
equity index/value
fills count/order/side/qty/price/time/fee/slippage/tag
positions snapshot
realized history
benchmark
metrics
config semantics
```

不比较旧 Python 类名、pickle bytes 或内部私有属性。

## 6. 测试计划

### 6.1 Indicator tests

- P1 23 项 golden 全覆盖。
- 空输入、单点、窗口大于数据、常量序列。
- timezone-aware index。
- float32/float64 normalization。
- NaN、inf 输入策略。
- optional PyWavelets 有/无两种路径明确，不静默改变算法版本。

### 6.2 MarketDataset tests

- 多标的/多 timeframe。
- 重复和乱序拒绝。
- OHLC 不变量错误。
- Arrow round-trip。
- RecordBatch stream 不一次性读完整输入的单元验证。

### 6.3 Backtest unit tests

- Order 状态机。
- Portfolio 守恒。
- long/short 限制。
- 8 cost models。
- 7 fill models。
- 2 execution models。
- same-bar intrabar SL/TP 优先级。
- lookahead 防护。
- 多 timeframe 可见性。
- 局部 RNG 不影响进程全局 numpy 状态。

### 6.4 Golden parity

- P1 代表回测逐字段对比。
- PAXG 代表策略 x 费率对比。
- Windows/Linux 在固定数据下核心结果一致。
- 重复运行固定 seed 一致。

### 6.5 Metamorphic/property

- 正成本收益不高于零成本同一成交路径。
- 订单/fill 时间因果性。
- 不允许现金与权益出现未解释 NaN。
- 价格比例缩放下百分比收益不变。
- 不允许未授权负持仓。

### 6.6 Dependency tests

- Kernel 不导入 FastAPI/httpx/SQLAlchemy/Redis/matplotlib。
- Kernel 不导入旧 `stockstat` 包。
- Capability Executor 不依赖 Worker Agent。

## 7. 性能基线

记录而非过早优化：

- 23 指标相对旧实现耗时和峰值内存。
- 代表回测相对旧实现耗时。
- Arrow decode/encode 开销。
- 多 timeframe Universe 构建成本。
- Result Artifact 大小。

退出门禁建议：核心算法在相同单进程条件下无超过 20% 的未解释回退；若为了更强校验有开销，必须有 benchmark 和可配置验证级别。

## 10. 风险与处理

| 风险 | 处理 |
|---|---|
| “完全重构”导致金融语义误改 | 以 golden 和不变量为先，不强行事件驱动 |
| Result schema 遗漏旧字段 | 逐个旧 BacktestResult 使用点审计 |
| pandas 版本引起 dtype 漂移 | schema 显式 cast，fixture 记录版本 |
| 策略对象状态不可重建 | 使用 factory + config，禁止闭包 |
| optional 算法 fallback 结果不同 | 算法版本显式写 Manifest |

## 11. 退出条件

- Kernel 单独安装可运行。
- 23 指标通过 golden。
- 单次回测当前核心语义通过代表性 parity。
- Cost/Fill/Execution 完整迁移。
- Result 不依赖 pickle，Arrow/JSON round-trip 完整。
- Kernel 无服务依赖和 legacy import。
- 性能回退已量化并在门限内或有批准说明。
