# P4：持久 Dispatcher、金融 Planner 与可靠状态机

## 1. 阶段目标

P4 实现无金融算法依赖的持久 Dispatcher：接收 JobSpec、调用 Capability Planner、固定输入 Snapshot、持久化 Stage/WorkUnit/Attempt、提供状态/事件，并实现幂等与可恢复调度基础。

P4 不实现真实 Worker 进程；测试使用 scripted worker harness 直接调用 Lease Service。

## 2. 范围

- Dispatcher API Gateway/command/query service。
- SQLite Task Store adapter。
- PostgreSQL Task Store adapter。
- Job/Stage/WorkUnit/Attempt/Event/Outbox schema。
- Idempotency-Key。
- Capability Planner registry。
- 首批 Indicator/Backtest Planner。
- Snapshot Coordinator 对 P3 Storage 的 adapter。
- Ready scheduling index。
- Lease claim/start/renew/complete/fail/release。
- retry/backoff/deadline/cancel。
- Result Manifest finalize。
- SSE event source service interface；真实 HTTP SSE 可在 P6 接线。

## 10. 测试计划

### 10.1 State model

- 所有合法状态迁移。
- 非法回退拒绝。
- terminal immutability。
- revision/sequence 单调。

### 10.2 Submit/idempotency

- 并发相同 key 只创建一个 Job。
- 相同 key 同 request 返回同 Job。
- 相同 key 不同 request conflict。
- planner validation/rejection。
- planning crash 后可恢复/重试。

### 10.3 Planner golden

- Indicator 单/多 symbol plan。
- Backtest 单 WorkUnit plan。
- Dataset snapshot refs 固定。
- fan-out quota。
- plan digest 稳定。
- executor role 变化会改变 plan digest。

### 10.4 Lease race/model tests

- 两个 claimer 争同 WorkUnit。
- renew/expiry 边界。
- complete/expiry 竞争。
- generation 1 迟到 complete 在 generation 2 成功后拒绝。
- complete 响应丢失后重复。
- release 后重新 ready。
- stale worker session。

### 10.5 Cancel/deadline

- queued cancel。
- running cancel + renew action。
- complete 与 cancel race，按事务先后确定稳定规则。
- deadline 在 queued/running。
- Client wait timeout 不改变 Job。

### 10.6 Retry

- worker lost。
- transient storage/artifact error。
- numerical terminal failure。
- max attempts。
- backoff 持久化并跨重启。

### 10.7 Restart recovery

- accepted/planning/queued/running 各状态重启。
- 未过期 lease 仍可 renew。
- 已过期 lease 回收。
- Outbox 未发布事件重放。
- Result 已写但 terminal event 未发的恢复。

### 10.8 Store parity

SQLite/PostgreSQL 运行相同测试。PostgreSQL 并发测试不得 skip。

## 13. 风险与处理

| 风险 | 处理 |
|---|---|
| SQLite 条件更新/并发不同 | contract + PG 强制并发测试 |
| 状态机过多隐式分支 | transition table/model tests |
| Planner 调 Storage 超时造成悬挂 | planning lease/recovery marker |
| token 明文持久化 | 仅存 token hash，响应返回原 token |
| Event 与状态不一致 | 同事务 outbox/event |

## 14. 退出条件

- Job/Stage/WorkUnit/Attempt/Event 全持久化。
- SQLite/PG Store 合同通过。
- Indicator/Backtest Planner golden 通过。
- Lease/fencing 的竞争和迟到结果测试通过。
- Dispatcher 重启恢复测试通过。
- Dispatcher 未导入 Kernel/pandas。
- scripted E2E 可产生可查询 Result Manifest。
